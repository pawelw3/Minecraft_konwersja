## Podsumowanie strategiczne

### Pełna widoczność kodu (możliwe automatyczne skrypty migracji):
Dla tych modów możesz napisać skrypty konwersji NBT używając `nbtlib` (Python) lub MCEdit Unified:
- **Mekanism** ↔ **Mekanism** (1.7.10 → 1.18.2) - struktury podobne
- **Thermal Expansion** ↔ **Thermal Expansion** - kompatybilność CoFH
- **Big Reactors** ↔ **Extreme Reactors** (oba mają kod, podobne struktury Yellorium)
- **Applied Energistics 2** ↔ **AE2** - wspólne DNA kodu
- **ComputerCraft** ↔ **CC:Tweaked** - ten sam autor (Dan200), kompatybilność Lua
- **Storage Drawers** (oba wersje) - możliwa migracja zawartości
- **Supplementaries** - otwarty kod dla szyldów, sakiew, książek

### Kod otwarty, ale assety zamknięte (ARR):
Możesz analizować logikę, ale nie używać assetów:
- **Ars Nouveau** - możesz stworzyć converter rituali, ale własne tekstury muszą być oryginalne
- **Another Furniture** - logika mebli dostępna, modele nie
- **Iron's Spells** - API spelli dostępne, tekstury zamknięte

### Kod zamknięty (migracja ręczna lub przez schematy):
- **Xaero's/JourneyMap** - waypointy trzeba eksportować przez GUI, nie przez NBT
- **IC2** (1.7.10) → **Tech Reborn** (1.18.2) - Tech Reborn ma kod otwarty, IC2 nie - asymetria utrudnia konwersję
- **Thaumcraft** (1.7.10, zamknięty) → **Ars Nouveau** (otwarty) - brak możliwości automatycznego mapowania researchu
